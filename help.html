故障处理

问题处理
红色失败告警先打开 GitHub Actions，查看最新失败 workflow；不要继续加功能。
黄色 running代表 workflow 正在运行或排队，等待结束后页面会在 60 秒刷新中读取新状态。
黄色 stale代表 data/status.json 超过阈值未更新，检查自动同步流水线是否停跑。
打开网页 404确认 Pages workflow 成功部署，index.html 在仓库根目录，稍等缓存刷新。
页面还是旧版按 Ctrl + F5 强制刷新，或等 GitHub Pages 缓存更新。
状态没变确认 data/status.json 已被最新 Actions 同步提交；也可点击“立即刷新状态”。
想连接 MT5只能先做只读状态，不开放下单。

安全边界

  • 不把密钥放进公开仓库。
  • 不在公开页里放主仓库代码。
  • 不开放交易按钮。
  • 不让网页直接控制实盘。
  • 不把 GitHub Issue / Actions 结果转成交易动作。
